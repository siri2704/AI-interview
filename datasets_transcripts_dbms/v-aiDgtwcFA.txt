dbms anomalies anomalies and databases refer to problems that occur when data is inserted updated or deleted in a way that violates the Integrity of the data there are several types of anomalies that can occur in a database insert anomaly this occurs when data can't be inserted into the database because it violates the rules of the data model for example a newly registered student cannot be inserted in the table because he does not have a roll number yet and the table does not accept null values update anomaly this occurs when an update to the database leads to an inconsistent or unexpected result for example if a student's address is updated in one place in the database but not in another the data becomes inconsistent delete anomaly this occurs when a deletion from the database leads to a loss of important information for example if the college closes the BCA Department then removing data from those rows will lead to a loss of students information
