asset properties in SQL transactions number one atomicity this property states that all the changes to the database should happen at once or none should happen at all for example in a bank transaction that involves money transfer both debit and credit should happen or none of them should happen at all number two consistency this property states that the data in the database should remain consistent before and after the transaction for example total amount in the two bank accounts involved in a bank transaction should remain same before and after the bank transaction took place number three isolation this property ensures that two transactions happening concurrently don't know about each other consider for transactions T1 and T2 regardless of their order of execution the final outcome in the database is going to be the same number four durability this property ensures that the changes made to the database is going to persist and cannot be reversed even in case of system failure or crash please like and share it and subscribe to my YouTube channel
